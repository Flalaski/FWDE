# Decision Log

| Date | Decision | Rationale |
|------|----------|-----------|
| 2026-07-12 | Implemented multi-pass chain-effect collision resolution (5 passes with diminishing weights) | Single-pass collision resolution only propagated forces to direct neighbors. Multi-pass with probed positions allows velocity accumulated in pass 1 (A→B) to cascade to pass 2 (B→C), creating realistic chain reactions through window clusters. Pass weights: 0.40, 0.25, 0.17, 0.10, 0.08 for controlled total force. |
| 2026-07-12 | Expanded all parameter slider ranges 2-5x and added missing parameter overrides | Many parameters were at their max (RepulsionForce=5 capped at 5, PairSeparationBase=0.08 capped at 0.08, etc.) preventing further tuning. All ranges significantly expanded: RepulsionForce 0.01-20, PairSeparationBase 0.001-0.5, RepulsionImpulseScale 0.01-10, MaxSpeed 0.5-120, NoiseScale 100-20000, etc. Added overrides for previously uncovered params: SeedDiagonalStep, NoiseScale, NoiseInfluence, ManualWindowAlpha, etc. |
| 2026-07-12 | Wired Config["Damping"] into all physics loop damping factors; user config values adopted as new defaults | Previously hardcoded damping factors (0.994, 0.998, 0.99, 0.98, 0.95, 0.88) ignored the Config["Damping"] parameter. Now all damping uses Config["Damping"] with a slight electron-app boost. New defaults reflect user's tuned config: higher repulsion (5.0), zero damping (1.0), ultra-fast visual step (1ms), generous overlap tolerance (50). |
| 2025-08-02 | Redesigned FWDE physics system from center-attraction to overlap-based repulsion | The original center-attraction system was too forceful and noticeable. The new system only acts when windows actually overlap or are too close, providing gentle separation forces. This creates a more natural, subtle experience where windows maintain their positions unless there's a real need to adjust them. Also added user move detection to temporarily pause physics when someone manually positions a window. |
